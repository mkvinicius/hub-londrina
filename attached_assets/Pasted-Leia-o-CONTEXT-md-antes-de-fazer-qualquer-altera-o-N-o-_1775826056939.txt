## O que precisa ser feito

parâmetros numéricos via URL. Sem isso, inputs maliciosos
como /negocio/abc ou /negocio/99999999999999 podem causar

## 1. CRIAR HELPER DE VALIDAÇÃO

Criar arquivo api-server/src/middleware/validateId.ts:

```typescript
import { Request, Response, NextFunction } from 'express';

// Valida e retorna ID numérico seguro
// Retorna null se inválido
export function parseId(value: string | undefined): number | null {
  if (!value) return null;
  const id = parseInt(value, 10);
  if (isNaN(id) || id <= 0 || id > 2147483647) return null; // max int4 PostgreSQL
  return id;
}

// Middleware para rotas com :id
export function validateId(req: Request, res: Response, next: NextFunction) {
  const id = parseId(req.params.id);
  if (id === null) {
    return res.status(400).json({
      error: 'ID inválido.',
      code: 'INVALID_ID'
    });
  }
  req.params.id = String(id); // normaliza
  next();
}

// Middleware para paginação
export function validatePagination(req: Request, res: Response, next: NextFunction) {
  const page = parseInt(req.query.page as string || '1', 10);
  const limit = parseInt(req.query.limit as string || '12', 10);

  if (isNaN(page) || page < 1) req.query.page = '1';
  else if (page > 10000) req.query.page = '10000';

  if (isNaN(limit) || limit < 1) req.query.limit = '12';
  else if (limit > 100) req.query.limit = '100'; // máximo por request

  next();
}
```

## 3. PROTEÇÃO ADICIONAL — IDs SEQUENCIAIS

Os IDs de negócio são sequenciais (/negocio/1, /negocio/2).
Isso expõe o tamanho da base mas não é um risco de segurança
direto. Para dificultar enumeração no futuro, adicionar
header de rate limit nas rotas públicas de negócio:

Criar limitador específico em rateLimiter.ts:
```typescript
export const businessViewLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 minuto
  max: 60,             // 60 visualizações por minuto por IP
  message: { error: 'Muitas requisições. Tente novamente em breve.', code: 'TOO_MANY_REQUESTS' },
  standardHeaders: true,
  legacyHeaders: false,
});
```

Aplicar em:
```typescript
router.get('/:id', businessViewLimiter, validateId, async (req, res) => { ... });
```
