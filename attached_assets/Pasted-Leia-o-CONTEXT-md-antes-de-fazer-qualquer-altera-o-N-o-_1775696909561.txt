Restaurantes. A busca precisa ser mais inteligente.

Correções no backend (GET /api/businesses e /api/search):

1. Busca case-insensitive e sem acento:
   - Normalizar o termo buscado (remover acentos, lowercase)
   - Comparar contra: name, description, tags, categorySlug,
     neighborhood, region

2. Busca parcial:
   - "pizza" deve retornar "Pizzaria do Rei" e negócios
     com tag "pizza"
   - Usar ILIKE no PostgreSQL:
     WHERE name ILIKE '%pizza%'
     OR description ILIKE '%pizza%'
     OR tags::text ILIKE '%pizza%'
     OR categorySlug ILIKE '%pizza%'

3. Busca por sinônimos de categoria:

   - "mercado" ou "supermercado" → categorySlug "mercados"
   - "café" ou "cafeteria" → categorySlug "cafeterias"
   - "pet" ou "veterinário" ou "veterinaria" → categorySlug "pet-shops"
   - "farmácia" ou "farmacia" ou "drogaria" → categorySlug "farmacias"
   - "padaria" ou "pão" ou "pao" → categorySlug "padarias"
   - "saúde" ou "saude" ou "clínica" ou "clinica" → categorySlug "saude"
   - "serviço" ou "servico" ou "mecânica" ou "mecanica" → categorySlug "servicos"

4. Busca na home (barra de pesquisa principal):
   - Deve usar as mesmas regras acima

### PROBLEMA 6 — Relevância paga na busca

Implementar o sistema de impulsionamento que afeta
a posição na busca.

#### Backend — campo novo no banco
Adicionar em businesses via migration Drizzle:
- boostedUntil: timestamp nullable
  (data até quando o negócio está impulsionado)

#### Ordenação da busca (aplicar nesta ordem exata):
1. Negócios com boostedUntil > NOW() — embaralhados
   aleatoriamente entre si (cada request tem ordem diferente)
2. Plano premium — ordenados por rating DESC
3. Plano destaque — ordenados por rating DESC
4. Plano free — ordenados por rating DESC

#### Como embaralhar os impulsionados:
No SQL, usar ORDER BY RANDOM() apenas para os
impulsionados, depois concatenar com o resto ordenado.

Ou em JavaScript após a query:
const boosted = results.filter(b => b.isBoosted);
const rest = results.filter(b => !b.isBoosted);
const shuffledBoosted = boosted.sort(() => Math.random() - 0.5);
return [...shuffledBoosted, ...rest];

#### No painel admin:
Adicionar campo "Impulsionar até" na gestão de cada negócio:
- Input de data: o admin define até quando o negócio
  fica impulsionado
- PATCH /api/admin/businesses/:id aceita { boostedUntil }
- Na tabela de negócios do admin, mostrar badge "BOOST"
  em vermelho para negócios com boostedUntil > agora

#### No painel do lojista (Premium):
- Card informativo mostrando se há boost ativo e até quando
- Por ora sem self-service de compra — o admin ativa manualmente
  (pagamento será implementado depois)
