## PROBLEMA 1 — `/api/businesses` retorna tudo sem autenticação

O endpoint `GET /api/businesses` está retornando 24 negócios completos com dados pessoais sensíveis (ownerName, ownerEmail, ownerPhone, cnpj, razaoSocial, lat/lng, clicks, planFrozen, etc.) SEM precisar de token.

### O que fazer:

PASSO 1: Encontre onde está definido o endpoint `/api/businesses` no backend. Provavelmente em `routes/api.ts` ou `routes/business.ts`.

PASSO 2: Encontre o middleware de autenticação JWT existente. Provavelmente em `middleware/auth.ts`. Se não existir, crie.

PASSO 3: Aplique o middleware de autenticação no endpoint `/api/businesses`. Exemplo:
javascript
router.get('/businesses', authenticateJWT, businessController.list);

PASSO 4: Crie um endpoint PÚBLICO separado para dados do site:
javascript
// Este endpoint é PÚBLICO — sem auth
router.get('/public/businesses', businessController.listPublic);

PASSO 5: Na função `listPublic`, crie uma função `sanitizeBusiness` que remove campos sensíveis:
javascript
function sanitizeBusiness(business) {
