## O que precisa ser feito

Criar o cadastro público de lojistas em /cadastro.
Qualquer empresário deve conseguir se cadastrar sozinho,
sem depender do admin para criar a conta manualmente.

---

## 1. BACKEND — POST /api/auth/register

Body:
{
  name: string,           -- nome do responsável
  email: string,          -- email de login
  password: string,       -- mínimo 8 caracteres
  businessName: string,   -- nome do negócio
  cnpj: string,           -- formato XX.XXX.XXX/XXXX-XX
  phone: string,          -- telefone com DDD
  categorySlug: string,   -- slug da categoria
  zone: string,           -- norte | sul | leste | oeste | centro
  cep: string             -- CEP do negócio
}

Validações em ordem — parar na primeira falha:

1. Email único:
   SELECT * FROM business_users WHERE email = ?
   Se encontrar → 400 { error: 'Email já cadastrado', code: 'EMAIL_DUPLICATE' }

2. CNPJ único:
   SELECT * FROM businesses WHERE cnpj = ?
   Se encontrar → 400 { error: 'CNPJ já cadastrado', code: 'CNPJ_DUPLICATE' }

3. Telefone único:
   SELECT * FROM businesses WHERE phone = ?
   Se encontrar → 400 { error: 'Telefone já cadastrado', code: 'PHONE_DUPLICATE' }

4. Validação CNPJ via ReceitaWS:
   GET https://receitaws.com.br/v1/cnpj/{cnpj_sem_mascara}
   Se status === 'ERROR' ou situacao !== 'ATIVA':
   → 400 { error: 'CNPJ inválido ou empresa inativa', code: 'CNPJ_INVALID' }
   Se a API falhar (timeout/erro): continuar sem bloquear
   (não punir o lojista por instabilidade da API externa)

5. Senha mínimo 8 caracteres:
   → 400 { error: 'Senha deve ter mínimo 8 caracteres' }

Se todas as validações passarem:

a) Criar registro em businesses:
   - name: businessName
   - cnpj, phone, zone, categorySlug
   - ownerName: name
   - ownerEmail: email
   - planType: 'free'
   - isVisible: false  ← INVISÍVEL até admin aprovar
   - status: 'pending' ← novo campo, adicionar via migration
   - city: 'Londrina', state: 'PR'
   - cep: cep

b) Buscar endereço do CEP via ViaCEP:
   GET https://viacep.com.br/ws/{cep}/json/
   Se retornar dados: salvar street, neighborhood

c) Criar registro em business_users:
   - businessId: id do negócio criado
   - email: email
   - passwordHash: bcrypt.hash(password, 10)

d) Retornar:
   201 {
     message: 'Cadastro recebido! Nossa equipe vai revisar em até 24h.',
     businessId: id
   }

## 3. FRONTEND — PÁGINA /cadastro

Formulário de 4 passos com barra de progresso no topo.

### Passo 1 — Sua conta
- Nome completo (obrigatório)
- Email (obrigatório, validar formato)
- Senha (obrigatório, mínimo 8 caracteres, mostrar força)
- Confirmar senha (deve ser igual)

### Passo 2 — Seu negócio
- Nome do negócio (obrigatório)
- CNPJ (obrigatório, máscara XX.XXX.XXX/XXXX-XX)
  → ao sair do campo: chamar GET /api/auth/validate-cnpj?cnpj=X
  → mostrar feedback: spinner → "CNPJ válido ✓" (verde) ou "CNPJ inválido ✗" (vermelho)
- Telefone com DDD (obrigatório, máscara (XX) XXXXX-XXXX)
- Categoria (select com as categorias do banco)
- Zona (select estilizado com cores por zona:
    Norte #3d7a28, Sul #2563eb, Leste #d97706,
    Oeste #7c3aed, Centro #dc2626)

### Passo 3 — Endereço
- CEP (máscara XXXXX-XXX)
  → ao sair do campo: chamar GET /api/lojista/cep/:cep (já existe)
  → preencher automaticamente: rua, bairro
  → mostrar feedback: "Endereço encontrado ✓"
- Rua (preenchido automaticamente, editável)
- Número (obrigatório)
- Bairro (preenchido automaticamente, editável)
- Cidade: "Londrina" (fixo, não editável)
- Estado: "PR" (fixo, não editável)

### Passo 4 — Confirmação
- Resumo dos dados:
  Nome do negócio, CNPJ, Categoria, Zona, Endereço
- Checkbox obrigatório: "Li e aceito os Termos de Uso"
- Botão: "Cadastrar meu negócio" (laranja, estilo padrão da plataforma)
- Estado de loading durante o envio

### Após sucesso:
Tela de confirmação (sem redirecionar para /lojista ainda):
- Ícone de check verde grande
- Título: "Cadastro recebido!"
- Texto: "Nossa equipe vai analisar seu cadastro em até 24h.
  Você receberá uma confirmação no email [email informado]."
- Link: "Voltar para a página inicial"

### Tratamento de erros:
- EMAIL_DUPLICATE → "Este email já possui uma conta. Faça login."
- CNPJ_DUPLICATE → "Este CNPJ já está cadastrado na plataforma."
- PHONE_DUPLICATE → "Este telefone já está cadastrado."
- CNPJ_INVALID → "CNPJ inválido ou empresa inativa na Receita Federal."
- Erro genérico → "Erro ao processar cadastro. Tente novamente."
