ITEM 4.1 — Global error handler em app.ts
Em artifacts/api-server/src/app.ts adicione no final, antes do export, um middleware de erro global:
app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
  req.log.error({ err, path: req.path, method: req.method }, "Unhandled error");
  if (res.headersSent) return next(err);
  res.status(500).json({ error: "Erro interno do servidor. Tente novamente." });
});
Também adicione handler para promises não capturadas no index.ts:
process.on("unhandledRejection", (reason) => logger.error({ reason }, "Unhandled rejection"));
process.on("uncaughtException", (err) => { logger.error({ err }, "Uncaught exception"); process.exit(1); });

ITEM 4.2 — Audit log (tabela admin_actions)
Crie lib/db/src/schema/admin-actions.ts:
export const adminActionsTable = pgTable("admin_actions", {
  id: serial("id").primaryKey(),
  adminId: integer("admin_id").notNull(),
  action: text("action").notNull(),
  targetType: text("target_type").notNull(),
  targetId: integer("target_id"),
  details: text("details"),
  ip: text("ip"),
  createdAt: timestamp("created_at").defaultNow(),
});
Adicione export em lib/db/src/schema/index.ts. Rode drizzle-kit push.
Crie função helper em artifacts/api-server/src/lib/audit.ts:
export async function logAdminAction(adminId: number, action: string, targetType: string, targetId?: number, details?: string, ip?: string) {
  await db.insert(adminActionsTable).values({ adminId, action, targetType, targetId, details, ip });
}
Use essa função nos endpoints admin mais críticos: aprovar/rejeitar cadastro, mudar plano, aprovar/rejeitar banner, aprovar/rejeitar documentação, deletar negócio, criar/deletar boost.
Adicione endpoint GET /api/admin/audit-log com filtros ?targetType=&adminId=&limit=50.
Adicione página AdminAuditLog.tsx no painel admin com tabela paginada e filtros.
Adicione link "Audit Log" no menu lateral do admin.

ITEM 4.5 — Sentry (free tier)
Instale: cd ~/workspace/artifacts/api-server && pnpm add @sentry/node
Em artifacts/api-server/src/index.ts inicialize Sentry antes de qualquer import:
import * as Sentry from "@sentry/node";
if (process.env.SENTRY_DSN) {
  Sentry.init({ dsn: process.env.SENTRY_DSN, environment: process.env.NODE_ENV || "production" });
}
Adicione Sentry.captureException(err) no global error handler do item 4.1.
Se SENTRY_DSN não estiver configurado nos Secrets, o sistema funciona normalmente sem Sentry (graceful degradation).
Adicione SENTRY_DSN como secret vazio por enquanto — pode ser preenchido quando criar conta no sentry.io.
