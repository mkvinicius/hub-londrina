## PARTE 2 — BACKEND: rotas do lojista

### Autenticação do lojista
POST /api/lojista/login
  Body: { email, password }
  → valida contra business_users
  → retorna JWT com { businessId, email }
  → token expira em 7 dias
  → guardar no localStorage como "hub_lojista_token"

### Perfil
GET  /api/lojista/profile
  → retorna todos os dados do negócio vinculado ao token

PATCH /api/lojista/profile
  Body: qualquer campo editável de businesses
  (name, description, phone, whatsapp, hours, cnpj, ownerName,
   ownerPhone, cep, street, number, neighborhood, instagram,
   website, paymentMethods, tags, zone, category)
  → salva no banco
  → retorna negócio atualizado

### Upload de imagens
POST /api/lojista/upload/logo
  → recebe multipart/form-data com campo "file"
  → salva em /public/uploads/logos/
  → atualiza logoUrl no banco
  → retorna { logoUrl }

POST /api/lojista/upload/banner
  → mesmo fluxo, salva em /public/uploads/banners/
  → atualiza bannerUrl

POST /api/lojista/upload/photo
  → salva em /public/uploads/photos/
  → adiciona URL ao array photos[]
  → respeita limite por plano (1 free, 10 destaque, ilimitado premium)
  → retorna { photoUrl, totalPhotos }

DELETE /api/lojista/photos/:index
  → remove foto do array pelo índice

### Geocodificação por CEP
GET /api/lojista/cep/:cep
  → chama https://viacep.com.br/ws/{cep}/json/
  → retorna { street, neighborhood, city, state }
  → NÃO salva automaticamente — só retorna os dados para preencher o form

### Coordenadas
PATCH /api/lojista/location
  Body: { cep, street, number, neighborhood }
  → chama API de geocodificação para obter lat/lng
  → usar https://nominatim.openstreetmap.org/search com parâmetros:
    q = "{street}, {number}, {neighborhood}, Londrina, PR, Brasil"
    format=json&limit=1
  → salva lat, lng, cep, street, number, neighborhood no banco
  → retorna { lat, lng }

### Produtos (vitrine)
GET    /api/lojista/products
POST   /api/lojista/products       Body: { name, description, price, mediaUrl, mediaType, whatsappLink }
PATCH  /api/lojista/products/:id   Body: qualquer campo
DELETE /api/lojista/products/:id
PATCH  /api/lojista/products/reorder  Body: [{ id, sortOrder }]

### Métricas
GET /api/lojista/metrics
  → retorna:
  {
    totalClicks: number,
    whatsappClicks: number,
    phoneClicks: number,
    profileViews: number,
    last30Days: { date: string, clicks: number }[]  -- série temporal
  }

### Senha
PATCH /api/lojista/password
  Body: { currentPassword, newPassword }
  → valida senha atual, salva nova com bcrypt
